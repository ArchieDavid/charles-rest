General


Like already mentioned, you interact with this service via Github issue comments. The code behind it can be found here. In short, there is a job that uses the Github API to interact with the @charlesmike account. Every minute, the notifications are checked and actions are taken according to each notification.

The crawling, fetching and storing of web-content logic is in this library. You can think of the library as the "engine" of everything, while the chatbot and the code behind it are only the "front end", the part which interacts with the user. Of course, it is reusable and you can use it in your own project if you want to. The crawling and fetching of content can be used directly, while you might want to define your own Repository


Commands


For now, 3 types of commands are "understood": hello, index site, and index page. We say types because regex rules are used and so, for instance, "hi" and "hello" or "hi there!" will be understood as salutations.

  • hello will trigger a salutation from the bot, and it will reply explaining what it can offer and on what conditions. Give it a try: say @charlesmike helo there!! in an issue comment
  • index site will trigger the crawling and indexing of the whole website and can be triggered by saying @charlesmike pls index or simply @charlesmike index
  • index page will trigger the crawling and indexing of a single page of the website and can be triggered by saying something like @charlesmike pls index [this](http://link-to-page.com) - here Markdown styling is used, to pretty format the Github comment and also make the text easely parasable.

Any Github user, in any Issue, can say hi to the bot and it will answer, but only the repository owner can trigger index commands. Owner means that the repo is under his/her name and it is not a fork. If the repo is under an organization, then the user must be admin of that organization. If the ownership condition is not met, the bot will answer explaining that this condition is not met and will not take any further action.

Index and Search


The key points about the indexing process are:

  • Runs only on command. It is no preprogrammed and does not run periodically!
  • You can index the whole website or just particular pages, one page at a time.
  • Once the index command is run your content becomes searcheable almost instantly.
  • The data is not altered in any way! We index it as we find it.
  • Your data is safe with us, we do not sell it or make it public in any way other than the search endpoint(s)!
  • You should only index public data that could be accessible through the web, by navigating through your site. In other words, do not try to index something you wouldn't like to see published on your site!
    •
  • The indexes reside in the cloud, on Amazon EC2 instance(s).

The key points about the searching process are:

  • You have to edit your site a little, just the first time (ideally, only add a script we provide).
  • Once an index command is finished, your search will see it almost in real time.
  • Relies on Javascript. It won't work if the javascript is disabled in the browser!
  • Should work fine on all modern browsers and mobile. We don't care about Internet Explorer; you should catch up with the world if you do.

About @charlesmike

Charles Michael is a Github chatbot which crawls and indexes your website's content making it searchable. You simply have to add a JS script to your page to render the search box.
